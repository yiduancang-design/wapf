# 单场深度拆解模块

用于用户要求“单场深度拆解”“单场比赛分析”“一场讲透”“逻辑自洽”“让用户听着有道理”的足球口播任务。语料只提供结构、节奏和表达方法，不作为事实来源。

## 核心目标

单场深度稿不是堆资料，而是把一个判断讲成闭环：

`结论 -> 争议点 -> 证据链 -> 战术翻译 -> 风险承认 -> 最终方向`

每个结论后必须回答两句话：

1. `凭什么这么说？`
2. `这会怎样影响比赛？`

## 标准结构

### 1. 开场 10 秒

要求：前 20 字内出现比赛对象或核心判断。

可选形态：

- 反常识：`这场大家都在看X，我反而更担心Y。`
- 高估/低估：`X队最容易被高估的地方，不是实力，而是比赛节奏。`
- 单变量：`这场只看一个变量：X能不能把Y拖进自己的节奏。`
- 回溯兑现：`上一场我们聊到的X已经验证，今天这场重点看Y。`

避免：夸张收益承诺、阴谋论、暗语、低俗攻击词。

### 2. 背景定调

用 2 至 4 句说明比赛为什么重要：

- 小组/淘汰赛形势
- 首轮或上一场结果带来的心理变化
- 主客/场地/旅途/体能变量
- 市场或舆论的普遍预期，但不要写成阴谋

写法：

`表面上这是A强B弱，但这场真正的变量不是名气，而是X。`

### 3. 目标队拆解

围绕用户给出的方向选择一方作为主线，不平均分配。

优先展开 3 个论点：

- 进攻来源：边路、肋部、中路渗透、定位球、反击、前场个人能力
- 防守结构：高位压迫、低位阵型、中卫转身、防空、禁区保护
- 状态和战意：上一场表现、出线压力、轮换、伤停、旅途和主场

每个论点按 `概括句 + 证据 + 比赛意义` 写。

示例骨架：

`第一，X的优势不是控球，而是边路第一下提速。上一场他们多次通过左路制造纵深，这说明只要对手边后卫被压住，X就能把比赛打成自己熟悉的节奏。`

### 4. 对手镜像对比

不要只骂弱点，要说明弱点如何被主线方利用：

- 对手进攻慢 -> 主线方能提前落位
- 对手边路回防差 -> 主线方能连续冲击
- 对手依赖门将/远射 -> 威胁转化不稳
- 对手体能掉速 -> 下半场更容易出现失误

写法：

`对手不是没有优点，但他们的优点刚好碰到X的强项会打折。`

### 5. 关键变量

单场深度稿必须给出一个真正能改变比赛走势的变量。优先选择：

- 早段是否进球
- 边路能否形成一对一
- 低位防守能否撑过 60 分钟
- 核心球员是否首发或状态是否完整
- 主场/旅途/天气是否影响体能
- 定位球是否成为破局点

变量必须可赛后验证。

### 6. 风险承认

专业感来自敢写反方风险。每篇至少一句：

`这场最大的风险是X，如果发生，比分可能不会拉开。`

不要把风险写成自我否定，而是作为判断边界。

### 7. 总结和方向

结尾必须落到用户给出的方向，不改变原预测。

结构：

`所以我的判断很明确：方向X。比分参考A、B。视频留着，赛后回来按三个点复盘：X、Y、Z。`

## 逻辑自洽检查

输出前逐项检查：

- 是否存在“前面说A进攻差，后面又说A会主动压制”的矛盾？
- 是否把舆论热度、盘口、水位直接当成赛果必然证据？
- 是否每个核心结论都有至少一个事实锚点或战术理由？
- 是否解释了“数据/场面”怎样转化为比赛走势？
- 是否保留一个反方风险？
- 是否没有未经核验的最新伤停、排名、赔率或引语？

## 常见扣分点

- 只说“实力强”“状态好”，没有说明强在哪里。
- 把两队写成互相独立的介绍，没有克制关系。
- 数据很多但不翻译成比赛画面。
- 全文只唱一个方向，没有风险边界。
- 开头很炸，正文证据接不住。
- 结尾只写“拭目以待”，没有复盘钩子。
