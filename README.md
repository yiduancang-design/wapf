# 文案评分优化 · WAPF

> 足球短视频口播文案的 **评分诊断 + 二次优化 + 单场深度拆解 + 下粉话术** 技能备份仓库。
>
> 本仓库用于保存 `$wenan` 技能的完整工作流说明、规则文件和语料引用，目标是：换电脑后可以下载恢复，后续版本也能继续同步到 GitHub。

---

## 它解决什么

足球口播文案最怕三件事：**没钩子、没证据、改完没有二次评分**。文案评分优化会先判断原稿质量，再按足球短视频逻辑做二次优化：

- **评分诊断**：按传播力、足球专业度、口播节奏、信息密度与可信度四个维度打分。
- **可验证性检查**：没有可赛后验证判断的稿子会被限制分数，避免空泛口号。
- **二次优化**：保留原预测方向，只增强钩子、补证据、调节奏、加复盘钩子。
- **二次复评**：优化稿必须重新评分，并对比改稿前后的提升。
- **单场深拆**：一场比赛按“结论 -> 证据 -> 战术翻译 -> 风险 -> 方向”形成闭环。
- **下粉话术**：只做平台内关注、收藏、评论、直播预约和赛后复盘引导，过滤收益承诺和站外导流风险。

> 重要：足球数据、伤停、赔率、赛程、身价和历史战绩都必须重新核验；语料只用于学习结构和节奏，不作为事实来源。

## 工作流程

### 任务一：已有文案评分 + 改稿

1. 读取原文，确认比赛、方向和预测立场。
2. 联网核验关键事实：赛程、阵容、伤停、近期战绩、赔率/市场信息等。
3. 按四维体系打分，给出不超过 3 条主要扣分理由。
4. 识别结构类型：单场深拆、多场串讲、反常识切入、数据驱动、叙事情怀等。
5. 按原方向改稿，不改变用户预测判断。
6. 优化稿自检，低于 8.0 分则继续迭代。
7. 输出完整优化稿。
8. 对优化稿做二次评分，并给出改稿前后对照。

### 任务二：生成新文案

1. 明确赛事、方向、结构类型和口播时长。
2. 联网检索最新情报。
3. 选择冷门但与比赛走势强绑定的切入点。
4. 按结构骨架生成完整文案。
5. 自评低于 8.0 分则重写。
6. 输出最终稿和自评报告。

### 任务三：单场深度拆解

单场稿必须形成闭环：

```text
结论 -> 争议点 -> 证据链 -> 战术翻译 -> 风险承认 -> 最终方向
```

每个核心结论都要回答：

```text
凭什么这么说？
这会怎样影响比赛？
```

---

## 质量铁律

- 优化稿必须达到 8.0 分以上。
- 必须输出二次评分，不能只给改稿不给分。
- 不能改变原预测方向。
- 不能编造数据、伤停、赔率、身价或历史战绩。
- 不能使用阴谋论、收益承诺、站外导流、暗语和虚假命中率。
- 结尾必须有可赛后复盘的钩子，避免“拭目以待”式软尾。
